# SpotiPlayGram

[![README in English](https://img.shields.io/badge/README-EN-blue)](README.md)

**SpotiPlayGram** — это Python-приложение, которое отображает текущий трек Spotify в Telegram-канале в реальном времени. Оно автоматически обновляет канал обложкой альбома, информацией о треке и опционально предоставляет ссылку на текст песни на Genius.

---

## 💡 Идея

Основная идея приложения — **добавить созданный канал в свой Telegram-профиль** (Настройки → Изменить профиль → Канал). При воспроизведении музыки в Spotify название канала автоматически меняется на **"Исполнитель – Название трека"**, а в сообщении отображается подпись **"Now Playing on Spotify"**.

Это создаёт эффект **полноценного статуса воспроизведения Spotify** прямо в вашем Telegram-профиле — другие пользователи видят, что вы сейчас слушаете, как в классическом Spotify-статусе.

- Автоматическая загрузка и отображение обложки альбома

- Автоудаление лишних сообщений для чистоты канала

### 📝 Интеграция с Genius
- Автоматический поиск ссылки на текст песни на Genius

- Использование Mistral AI для интеллектуального выбора лучшей ссылки
- Поддержка нескольких AI моделей с автоматическим переключением
- Настраиваемая приоритетная цепочка моделей

- Telegram-бот для администрирования
- Включение/выключение статуса воспроизведения
- Управление настройками Genius (паттерны версий, паттерны feat)
- Настройка моделей Mistral AI прямо из Telegram

### ⚙️ Дополнительные возможности
- Кроссплатформенная поддержка (Windows, Linux, macOS)
- Автоматическое создание виртуального окружения
- Опциональная ежедневная проверка обновлений зависимостей (в 00:00 UTC)

## 📋 Требования

- Python 3.10 или выше
- Аккаунт разработчика Spotify

## 🚀 Установка

### 1. Клонирование репозитория

```bash
git clone https://github.com/xsfeelyou/SpotiPlayGram.git
cd SpotiPlayGram
```

### 2. Настройка переменных окружения

Скопируйте файл `.env-example` в `.env` и заполните ваши учётные данные:

```bash
# Windows
copy .env-example .env

# Linux/macOS
cp .env-example .env
```

```env
# Spotify API
SPOTIFY_CLIENT_ID=ваш_spotify_client_id
SPOTIFY_CLIENT_SECRET=ваш_spotify_client_secret
SPOTIFY_REDIRECT_URI=http://localhost:8888/callback
SPOTIFY_LANGUAGE=RU  # Язык для запросов к Spotify API
SPOTIFY_REQUEST_TIMEOUT=5  # Таймаут запросов к Spotify API в секундах

# Telegram API
TG_API_ID=ваш_telegram_api_id
TG_API_HASH=ваш_telegram_api_hash
TG_BOT_TOKEN=ваш_токен_бота

# Telegram IDs
TG_CHANNEL_ID=1234567890
TG_ADMIN_USER_ID=ваш_user_id

# ID сообщений в канале
TG_MEDIA_MESSAGE_ID=2
TG_TEXT_MESSAGE_ID=3
TG_AUTO_DELETE_MESSAGES=True  # Автоматическое удаление служебных сообщений бота

# Genius API (опционально)
GENIUS_ACCESS_TOKEN=ваш_genius_токен
ENABLE_GENIUS=True  # Включить поиск текстов песен через Genius API
GENIUS_DETAILED_LOG=False  # Подробное логирование работы Genius

# Mistral AI (опционально)
MISTRAL_API_KEY=ваш_mistral_api_ключ
ENABLE_MISTRAL=True  # Включить Mistral AI для точного выбора URL текстов
MISTRAL_MODEL=mistral-large-latest  # Модель Mistral AI (по умолчанию)

# Повторные попытки авторизации
AUTH_RETRY_MAX_RETRIES=2  # Максимальное количество попыток авторизации
AUTH_RETRY_DELAY_BASE=1.0  # Базовая задержка между попытками авторизации в секундах

# Автообновление зависимостей
ENABLE_DEP_UPDATES=False  # Автоматическое обновление зависимостей при запуске

# Интервал обновления в секундах
UPDATE_INTERVAL=1  # Интервал опроса Spotify API в секундах
```

## 🔧 Настройка

### Настройка Spotify

1. Перейдите на [Spotify Developer Dashboard](https://developer.spotify.com/dashboard)
2. Создайте новое приложение
3. В настройках приложения добавьте Redirect URI: `http://localhost:8888/callback`
4. Скопируйте Client ID и Client Secret в `.env`

**Примечание об авторизации:** Spotify использует **OAuth 2.0 аутентификацию** — при первом запуске откроется браузер для входа в ваш аккаунт. **Авторизация через SSH невозможна**, так как нет доступа к графическому интерфейсу. Первую авторизацию необходимо выполнить на компьютере с графическим интерфейсом (Windows, Linux с GUI или macOS), а затем перенести файлы сессии на целевое устройство.

**Примечание о языке:** Выберите правильное значение для `SPOTIFY_LANGUAGE` в `.env` — оно должно соответствовать языку вашего Spotify-приложения. Этот параметр влияет на то, как отображаются имена исполнителей и названия треков в вашем Telegram-канале. Например, если имя исполнителя написано на нелатинском алфавите (кириллица, хангыль, иероглифы и т.д.), то при `SPOTIFY_LANGUAGE=EN` имя может отображаться латиницей, а при `SPOTIFY_LANGUAGE=RU` (или соответствующем коде языка) — в оригинальной письменности.

**Примечание об ограничениях:** Spotify Web API endpoint для статуса воспроизведения может требовать Premium-аккаунт (зависит от ограничений/политики Spotify).

### Настройка Telegram

1. Перейдите на [my.telegram.org](https://my.telegram.org)
2. Создайте новое приложение
3. Скопируйте API ID и API Hash в `.env`
4. Создайте бота через [@BotFather](https://t.me/BotFather)
5. Скопируйте токен бота в `.env`

#### Подготовка Telegram-канала

1. Создайте **публичный** Telegram-канал (публичный канал необходим для добавления в профиль)
2. Добавьте вашего бота как **администратора** с полными правами (это обязательно для работы бота)
3. Получите ID канала (`TG_CHANNEL_ID`) — используйте любой Telegram-клиент или бота, который показывает chat ID (например, перешлите сообщение из канала боту [@userinfobot](https://t.me/userinfobot), или получите ID программно через Telethon). Скопируйте полученный числовой ID (например, `1234567890`) в `.env`
4. Отправьте в канал **два** сообщения в следующем порядке:
   - **Первое сообщение** (`TG_MEDIA_MESSAGE_ID`) — отправьте любое изображение **со сжатием** (как фото, а не как файл) и добавьте к нему любой текст в подписи. Например, можно отправить `default.png` из папки `session/spotify/`. Это сообщение будет использоваться для обложки альбома. Telegram API позволяет заменять медиа только в сообщении, которое уже содержит изображение, поэтому обычное текстовое сообщение не подойдёт
   - **Второе сообщение** (`TG_TEXT_MESSAGE_ID`) — отправьте любой текст. Это сообщение будет использоваться для информации о треке
5. Получите ID каждого сообщения — нажмите правой кнопкой (или долгим нажатием) на сообщение и выберите "Копировать ссылку на сообщение". Ссылка выглядит как `https://t.me/имя_канала/123` — последнее число (`123`) и есть ID сообщения. Скопируйте ID первого сообщения (с картинкой) в `TG_MEDIA_MESSAGE_ID`, а ID второго (с текстом) в `TG_TEXT_MESSAGE_ID` в `.env`

#### Сессии Telegram

SpotiPlayGram использует **две отдельные Telethon-сессии**:

 - **User session** (`session/telegram/user.session*`) — требует полноценную авторизацию в Telegram (номер телефона и облачный пароль, если он установлен). Используется для действий, где нужен реальный пользователь
 - **Bot session** (`session/telegram/bot.session*`) — использует `TG_BOT_TOKEN`. Используется для панели управления (`/start`) и обновления канала

Для запуска на другом устройстве (ПК, сервер, VPS и т.д.) после первой авторизации скопируйте файлы `session/telegram/user.session*`, `session/telegram/bot.session*` и `session/spotify/.cache`.

### Настройка Genius (Опционально)

1. Перейдите на [Genius API Clients](https://genius.com/api-clients)
2. Создайте нового API клиента
3. Сгенерируйте access token
4. Скопируйте токен в `.env`

### Настройка Mistral AI (Опционально)

1. Перейдите на [Mistral AI Console](https://console.mistral.ai)
2. Создайте API ключ
3. Скопируйте ключ в `.env`

**Примечание о регистрации:** Регистрация в Mistral может потребовать номер телефона для верификации. На момент написания у Mistral есть бесплатный тариф (до ~1 млрд токенов в месяц), чего более чем достаточно для SpotiPlayGram (даже при прослушивании музыки 24/7 расход обычно сильно ниже ~2 млн токенов в месяц).

**Примечание о точности:** Без Mistral AI выбор ссылок на тексты песен из Genius будет менее точным. Встроенный алгоритм через Genius API хорошо работает в большинстве случаев, но в связке с Mistral AI значительно повышает точность.

**Примечание о кэшировании:** Mistral AI использует кэш результатов на 1 час для оптимизации производительности и снижения потребления токенов. Повторные запросы для одного трека будут обработаны мгновенно без дополнительных API вызовов.

**Примечание о выборе модели:** Сравнить бенчмарки моделей Mistral можно на [Artificial Analysis](https://artificialanalysis.ai/providers/mistral), чтобы выбрать более точную модель для подбора лучшей ссылки на текст песни из результатов поиска Genius. Однако учитывайте, что самые умные и тяжёлые модели со временем могут начать выдавать ошибку **HTTP 429** (`Service tier capacity exceeded for this model`) из-за лимитов бесплатного тарифа — если это происходит, переключитесь на более лёгкую модель.

## ▶️ Запуск приложения

**Windows:**
```bash
python start.py
```

Также можно запустить файл `start.py` двойным кликом или через контекстное меню "Открыть с помощью Python".

**Linux:**
```bash
python3 start.py
```

**Linux (графическая среда / GUI):** Рекомендуется запускать в обычной графической сессии (например, GNOME/KDE), потому что при первом запуске для авторизации Spotify должен открыться браузер.

**Linux (подключение по SSH):** Для непрерывной работы при подключении через SSH используйте `screen` или `tmux`, чтобы приложение продолжало работать после отключения от сессии:
```bash
# Использование screen
screen -S SpotiPlayGram
python3 start.py
# Нажмите Ctrl+A, затем D для отсоединения
# Для переподключения: screen -r SpotiPlayGram

# Использование tmux
tmux new -s SpotiPlayGram
python3 start.py
# Нажмите Ctrl+B, затем D для отсоединения
# Для переподключения: tmux attach -t SpotiPlayGram
```

**macOS:**
```bash
python3 start.py
```

При первом запуске:
- Автоматически создастся виртуальное окружение
- Установятся необходимые зависимости
- Вам будет предложено авторизоваться в Telegram (вход по номеру телефона, а также по облачному паролю, если установлен)
- Вам будет предложено авторизоваться в Spotify (откроется браузер)

Автообновление зависимостей опционально и управляется через `ENABLE_DEP_UPDATES` в `.env`.

### Работа 24/7

Если вы хотите, чтобы приложение работало непрерывно 24/7, у вас есть два варианта:
- **Держать приложение запущенным на ПК** — компьютер должен быть включён и приложение должно работать постоянно
- **Использовать свой сервер** — разверните приложение на VPS или выделенном сервере для бесперебойной работы

## 🎮 Команды бота

Отправьте `/start` вашему боту для доступа к панели управления:

- **📢 Playback status** — Включение/выключение обновлений статуса Spotify в канале Telegram
  - **▶️ Enable** — Включить обновления статуса воспроизведения
  - **⏹️ Disable** — Отключить обновления статуса воспроизведения
- **⚙️ Genius settings** — Настройка параметров поиска текстов песен через Genius API
  - **🤖 Mistral AI models** — Выбор AI моделей Mistral для интеллектуального подбора URL текста песни из результатов поиска Genius
    - **🔄** — Обновить список моделей Mistral
  - **Version patterns** — Управление списком слов/фраз для определения ремиксов, каверов, версий в названиях треков (например: "remix", "cover", "acoustic")
  - **Feat patterns** — Управление списком слов/фраз для определения featuring-исполнителей в названиях треков (например: "feat", "ft", "with")
    - **Add** — Добавить новый паттерн в выбранную категорию
    - **Delete** — Удалить существующий паттерн из выбранной категории
    - **Confirm add** — Подтвердить добавление введённых паттернов
    - **Confirm delete** — Подтвердить удаление выбранных паттернов

## 🔄 Как это работает

1. **Цикл опроса**: Приложение опрашивает Spotify API каждую секунду (настраивается)
2. **Определение трека**: При обнаружении нового трека получает полные метаданные
3. **Поиск текста**: Ищет трек на Genius с нечётким сопоставлением
4. **AI-выбор**: Mistral AI помогает Genius выбрать точную ссылку на текст песни
5. **Обновление Telegram**: Обновляет сообщения канала, название и фото
6. **Управление состоянием**: Сохраняет текущее состояние для избежания дублирования обновлений

## 🛠️ Устранение неполадок

### Spotify

**Ошибка авторизации / неверные учётные данные**
- Убедитесь, что `SPOTIFY_CLIENT_ID` и `SPOTIFY_CLIENT_SECRET` в `.env` совпадают с данными из [Spotify Developer Dashboard](https://developer.spotify.com/dashboard)
- Проверьте, что `SPOTIFY_REDIRECT_URI` в `.env` точно совпадает с Redirect URI, указанным в настройках приложения на Dashboard (по умолчанию `http://localhost:8888/callback`)

**Токен Spotify истёк / не удаётся получить данные о воспроизведении**
- Удалите файл `session/spotify/.cache` и перезапустите приложение — авторизация пройдёт заново через браузер
- Проверьте, что приложение в Spotify Developer Dashboard не удалено и находится в активном состоянии
- Если у вас бесплатный аккаунт Spotify, API статуса воспроизведения может быть недоступен — требуется Premium

### Telegram

**Ошибка авторизации / неверные учётные данные**
- Проверьте, что `TG_API_ID` и `TG_API_HASH` в `.env` совпадают с данными из [my.telegram.org](https://my.telegram.org)
- Убедитесь, что `TG_BOT_TOKEN` действителен — токен можно обновить через [@BotFather](https://t.me/BotFather)

**Канал не обновляется**
- Убедитесь, что бот добавлен в канал как **администратор** с полными правами
- Проверьте, что сообщения с ID `TG_MEDIA_MESSAGE_ID` и `TG_TEXT_MESSAGE_ID` действительно существуют в канале и не были удалены
- Убедитесь, что `TG_CHANNEL_ID` указан верно (числовой ID канала, например `1234567890`)

**Ошибки файлов сессии**
 - Если приложение не может подключиться к Telegram или выдаёт ошибки сессии, удалите файлы `session/telegram/user.session*` и `session/telegram/bot.session*`, затем перезапустите приложение для повторной авторизации
- Убедитесь, что другой экземпляр приложения не запущен одновременно — два процесса не могут использовать одну и ту же сессию

### Genius

**Поиск текстов не работает**
- Проверьте, что `GENIUS_ACCESS_TOKEN` в `.env` действителен — токен можно сгенерировать заново на [Genius API Clients](https://genius.com/api-clients)
- Убедитесь, что `ENABLE_GENIUS=True` в `.env`

### Mistral AI

**Mistral AI не отвечает / ошибки запросов**
- Проверьте, что `MISTRAL_API_KEY` в `.env` указан правильно и ключ активен в [Mistral AI Console](https://console.mistral.ai)
- Убедитесь, что `ENABLE_MISTRAL=True` в `.env`
- Если модель возвращает ошибку, убедитесь, что она доступна — модели периодически обновляются и могут быть переименованы или удалены. Актуальный список моделей можно посмотреть через бота (⚙️ Genius settings → 🤖 Mistral AI models)
- При частых таймаутах попробуйте переключиться на более лёгкую модель (например, `mistral-small-latest`)
